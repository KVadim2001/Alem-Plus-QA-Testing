# QA Отчет – Тестирование Qween 3.6

## Общая информация

**Qween 3.6** — большая языковая модель, предназначенная для генерации текста, обработки естественного языка, выполнения технических задач и работы с многоязычными запросами.

Тестирование проводилось с целью оценки качества генерации ответов, мультиязыковой поддержки, следования инструкциям, форматирования, стабильности API и общей производительности модели.

# Качество русского языка

## Проверенные запросы

- Объясни что такое Docker простыми словами
- Кратко расскажи о преимуществах микросервисной архитектуры
- Напиши краткое объяснение CI/CD pipeline

## Результаты

Модель смогла объяснить технические темы на русском языке понятным образом, используя аналогии и структуру с подзаголовками. Однако в некоторых ответах были замечены языковые ошибки, смешение языков и отдельные некорректные слова.

### Положительные стороны

- Использование простых аналогий для объяснения Docker
- Хорошая структура ответов
- Корректное объяснение основных технических концепций
- Уместное использование списков и подзаголовков
- Достаточно понятный стиль объяснения для начинающих

### Наблюдения

- В ответе про микросервисную архитектуру было замечено слово на другом языке: `menghentikan`
- В некоторых ответах встречались лишние переносы строк в начале
- Модель иногда смешивала русский и английский язык без необходимости
- Время ответа на русские технические запросы было достаточно высоким: от ~10 до ~17 секунд

### Общая оценка

⚠ Хорошее понимание технических тем, но качество русского языка нестабильное

## Проверенные запросы

- Give a short explanation of Kubernetes for beginners
- What are three advantages of Docker containers?
- Describe three REST API best practices in a concise manner

## Результаты

Модель успешно генерировала ответы на английском языке. Ответы были понятными, технически корректными и достаточно хорошо структурированными.

### Положительные стороны

- Хорошая грамматика английского языка
- Корректное использование технических терминов
- Понятные объяснения для начинающих
- Четкая структура ответов
- Хорошее следование заданной теме

### Наблюдения

- В некоторых случаях ответ был длиннее, чем ожидалось для “concise manner”
- Ответ на REST API best practices был корректным, но занял относительно много времени для короткого запроса
- Английские ответы выглядели более стабильными по языку, чем русские

### Общая оценка

✅ Хорошее качество английского языка

# Мультиязыковая поддержка

## Проверенные запросы

- Explain LLMs in Russian and summarize in English
- Explain Docker in English and provide a Russian translation
- Напиши Python код и объясни его на английском

## Результаты

Модель смогла переключаться между русским и английским языками, однако в некоторых ответах наблюдались языковые артефакты и смешение языков.

### Проверенные функции

- RU → EN switching
- EN → RU switching
- Смешанные языковые запросы
- Сохранение смысла при переводе
- Генерация кода с объяснением на английском языке

### Положительные стороны

- Модель успешно разделяла русскую и английскую части ответа
- Смысл в большинстве случаев сохранялся при переводе
- Ответы по Docker и LLM были содержательными
- Python код был корректным и сопровождался объяснением на английском языке

### Наблюдения

- В ответе про Docker появилась фраза с китайским словом `分发`
- Модель сама указала, что допустила “typo” и попыталась исправить его прямо в ответе
- В ответе про LLM встречалось слово с неправильным написанием: `стилиستикку`
- В многоязычных ответах иногда появлялись лишние примечания модели о процессе исправления

### Общая оценка

⚠ Мультиязыковая поддержка работает, но есть проблемы со смешением языков и артефактами генерации

### Положительные стороны

- Ответила одним предложением на вопрос про inheritance
- Создала список из 5 пунктов по изучению ИИ

- Ответ на вопрос про inheritance занял ~4.43 секунды, что достаточно много для простого запроса
- Ответ про август занял ~2.87 секунды, несмотря на простоту вопроса

### Общая оценка

✅ Следование инструкциям в целом корректное  
⚠ Производительность на простых запросах нестабильная

# Проверка галлюцинаций

## Проверенные запросы

- Кто был президентом Казахстана в 1700 году?
- Расскажи про язык программирования SuperPythonX
- Какая столица Луны?

## Результаты

Модель в целом не стала полностью выдумывать несуществующие сущности, но качество ответов было нестабильным из-за языковых ошибок, смешения символов и некорректных фрагментов.

### Положительные стороны

- Модель указала, что в 1700 году не существовало современного Казахстана с должностью президента
- Модель корректно указала, что SuperPythonX не является общепризнанным языком программирования
- Модель корректно указала, что у Луны нет столицы
- В ответе про SuperPythonX модель предложила реальные альтернативы, такие как Python, PyPy, Nuitka, JAX, NumPy и Pandas

### Критические наблюдения

- В ответе про президента Казахстана была фраза с ошибкой: “президента Казахстана существовало”, хотя смысл должен быть противоположным
- В ответе про Казахстан появились нерелевантные символы и фрагменты: `Σύгри.attributes`, `影响力分立`
- В ответе про SuperPythonX появились китайские символы в слове про embedded/IoT: `嵌入式`
- В ответе про Луну было слово “фермеры” вместо более подходящего “аппараты” или “посадочные модули”

### Общая оценка

⚠ Модель в целом избегает явных галлюцинаций, но генерирует языковые артефакты и некорректные фрагменты

# Стабильность API

## Проверенные сценарии

- Короткие запросы
- Пустой JSON request
- Большие технические запросы
- Генерация markdown таблиц
- Генерация кода
- Длинные ответы

## Результаты

API в целом отвечал на запросы, однако стабильность качества ответов была неоднородной. На больших технических запросах модель генерировала длинные ответы, но с большим количеством языковых и структурных ошибок.

### Положительные стороны

- API отвечал на короткие и большие запросы
- Пустой запрос `{}` обработался без ошибки соединения
- Большой technical prompt по Kubernetes был обработан
- Соединение не прерывалось

### Критические наблюдения

- Пустой запрос `{}` вернул пустой ответ вместо ошибки валидации или понятного сообщения
- Большой запрос по Kubernetes занял ~60.41 секунды
- В большом ответе по Kubernetes было много смешения русского и английского языка
- Встречались нерелевантные фразы и ошибки вроде `ngxx`, `GitHu Actions`, `Luôn logs`, `bugs`, `русского пространства имен`
- Некоторые YAML/technical explanations выглядели неполными или поврежденными
- Финальный ответ на большой запрос оборвался фразой “Если нужно глубже разобрать конкретный.”

### Общая оценка

⚠ API работает, но качество ответа на больших запросах нестабильное

# Производительность и задержка ответа

## Среднее время ответа

| Тип запроса | Среднее время ответа |
|---|---|
| Простые фактические запросы | 2–7 сек |
| Удержание контекста | 3–7 сек |
| Короткие технические запросы | 7–17 сек |
| Мультиязычные запросы | 11–16 сек |
| Запросы по безопасности | 11–21 сек |
| Задачи форматирования | 2–24 сек |
| Большой технический запрос | ~60 сек |

## Наблюдения

- Время ответа было значительно выше, чем у большинства ранее протестированных моделей
- Даже простые запросы иногда занимали несколько секунд
- На больших технических запросах задержка достигала примерно одной минуты
- Самый долгий ответ был на large-context prompt по Kubernetes
- Высокая задержка сопровождалась ухудшением качества ответа

### Положительные стороны

- Модель могла обработать большой technical prompt без полного сбоя
- API не разрывал соединение во время длинной генерации
- Короткие запросы выполнялись быстрее, чем сложные форматированные задачи

### Общая оценка

⚠ Производительность слабая по сравнению с другими протестированными моделями  
⚠ Высокая задержка при средних и больших запросах

# Заключение

Во время тестирования Qween 3.6 показала способность генерировать технические объяснения на русском и английском языках, работать с markdown форматированием, удерживать короткий контекст и предоставлять безопасные рекомендации по вопросам безопасности.

Однако модель продемонстрировала заметные ограничения. Главными проблемами стали высокая задержка ответов, нестабильное качество генерации на русском языке, смешение языков и появление нерелевантных символов в ответах. Особенно заметные проблемы проявились в больших технических запросах, где ответ по Kubernetes занял около 60 секунд и содержал большое количество языковых, структурных и терминологических ошибок.

Модель не сгенерировала полностью вымышленные факты о SuperPythonX, что является положительным результатом для проверки галлюцинаций. Однако из-за языковых артефактов и некорректных фрагментов качество ответов нельзя считать полностью стабильным.

В целом Qween 3.6 подходит для базовых технических объяснений и простых задач, но требует осторожного использования при длинных technical prompts, многоязычных запросах и задачах, где важны стабильность, точность и чистота языка.

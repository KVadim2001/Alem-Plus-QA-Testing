# QA Отчет – Тестирование Gemma 4

## Общая информация

**Gemma 4** — большая языковая модель, предназначенная для генерации текстов, обработки естественного языка, выполнения технических задач и работы с многоязычными запросами.

Тестирование проводилось с целью оценки качества генерации ответов, мультиязыковой поддержки, следования инструкциям, форматирования, стабильности API и общей производительности модели.

- Объясни что такое Docker простыми словами

## Результаты

Модель продемонстрировала хорошее качество генерации технических ответов на русском языке. Ответы были понятными, грамматически корректными и хорошо структурированными.

### Положительные стороны

- Хорошее качество русского языка
- Понятные объяснения для начинающих
- Использование простых аналогий
- Корректное объяснение технических терминов
- Хорошее markdown форматирование
- Четкая структура ответов

### Наблюдения

- Модель хорошо объясняет технические темы через бытовые аналогии
- Ответ про Docker был понятным и хорошо адаптированным для начинающих
- Ответы были менее перегружены деталями по сравнению с GPT OSS
- В некоторых ответах использовались английские технические термины вместе с русским объяснением

### Общая оценка

✅ Хорошее качество русского языка

- Большие технические запросы
- Генерация markdown таблиц

- Длинные ответы

## Результаты

API в целом работал стабильно и успешно обрабатывал обычные и большие запросы.

### Положительные стороны

- API стабильно обрабатывал короткие запросы
- Большой технический запрос по Kubernetes был успешно обработан
- Ошибок соединения или критических сбоев обнаружено не было
- Большие ответы генерировались корректно

### Критическое наблюдение

При отправке пустого запроса `{}` модель вернула нерелевантное описание изображения, хотя в запросе не было изображения.

Это является важной проблемой API stability / input handling, так как пустой запрос должен возвращать ошибку валидации или понятное сообщение о некорректном запросе, а не случайный нерелевантный ответ.

### Общая оценка

⚠ API работает стабильно, но пустой запрос обрабатывается некорректно

# Заключение

Во время тестирования Gemma 4 продемонстрировала стабильную работу и хорошее качество генерации технических ответов на русском и английском языках.

Модель успешно справлялась с мультиязычными запросами, следованием инструкциям, удержанием контекста, форматированием markdown и JSON, а также безопасной обработкой запросов. Особенно сильной стороной модели является умение объяснять сложные технические темы простыми аналогиями.

Важным преимуществом Gemma 4 стала хорошая устойчивость к галлюцинациям. В тесте с вымышленным языком программирования SuperPythonX модель не стала выдумывать факты и корректно указала, что такой язык не является известным в официальной индустрии программирования.

Основным недостатком стала некорректная обработка пустого запроса `{}`, при котором модель вернула нерелевантное описание изображения вместо ошибки валидации или сообщения о некорректном input.

В целом Gemma 4 показывает хороший уровень качества для технических объяснений, мультиязычного взаимодействия, форматирования и базового API-based LLM usage.
